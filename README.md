

# 需求最佳实践
大原则：切勿执行一次就进入下一个阶段，LLM是概率模型，多让Kiro自检，双向验证。

* 提前准备好需求文件，然后再创建spec，例如：project-architecture-design.md 是一个创建多模块maven工程的指导手册，请根据手册，初始化当前的项目
* 需求创建完成后，不要急于进入设计阶段，让Kiro检查需求是否与原始意图一致，是否有未澄清的内容，是否完全覆盖了原始意图，是否都准确的覆盖了原始意图，例如：请检查需求是否与原始意图一致，任何需要澄清的需求点，请务必与我确认，确保需求足够完备，足够准确。
* 让Kiro对设计自检，例如：
    * 请再次检查设计文档与需求文档是一致的，并且设计自身的内容内部没有冲突，且能完全并准确的覆盖需求。
    * 请检查设计中，是否有需要未提及的内容，属于过度设计的内容？如果有，请与我确认，是否补充和完善需求。
* 让Kiro检查任务列表是否与需求和设计一致，是否有超出设计范围的任务。另外，让Kiro明确每个任务的验证标准，并对任务进行验证。
